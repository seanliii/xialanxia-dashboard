# 内容自动驾驶系统

## 功能说明
填写内容策略（领域/风格/平台）→ 系统实时搜索真实素材（DuckDuckGo）→ AI评分筛选（时效性/实用性/标题吸引力）→ 生成7天内容日历 + 各平台适配草稿。支持 Markdown 导出。

## 在线使用
直接访问：https://seanliii.github.io/content-autopilot/

> ⚠️ 国内访问注意：实时搜索功能通过 DuckDuckGo，国内可能被墙，建议配合代理使用或直接使用离线模式（手动粘贴素材）

## 本地部署步骤

### Step 1：下载项目
```bash
git clone https://github.com/seanliii/content-autopilot.git
cd content-autopilot
```
**成功判断**：`ls` 显示 `index.html`，约26KB

### Step 2：本地预览
```bash
python3 -m http.server 8080
```
**成功判断**：页面显示内容策略配置区和日历预览区

### Step 3：配置内容策略
1. 填写「内容领域」（如：AI工具/个人理财/职场成长）
2. 填写「目标受众」（如：25-35岁互联网从业者）
3. 填写「风格关键词」（如：干货/实操/数据驱动）
4. 勾选「发布平台」（支持：微博/公众号/小红书/知乎/LinkedIn/Twitter）
5. 设置「周发布频率」（如：3次/周）

### Step 4：生成内容
1. 点击「搜索实时素材」→ 系统搜索相关领域近期内容
2. 每条素材显示 AI 评分（0-99），筛选≥70分的
3. 点击「生成7天日历」→ 自动规划发布节奏
4. 对每条内容点击「展开大纲」→ 生成各平台适配草稿
5. 点击「导出日历」→ 下载 Markdown/JSON 格式

**成功判断**：日历区域出现7天×多平台的内容规划表

## 踩坑记录
- DuckDuckGo 在国内访问不稳定，建议配合代理
- AI评分是基于规则的（关键词/时间/来源权威性），不是真正的语义理解
- 情感/个人故事类内容效果差，AI很难生成有温度的内容，这类内容建议只用大纲，正文自己写

## 适用场景
- 运营多个平台账号需要矩阵化生产
- 内容方向明确但不知道每天写什么
- 把内容生产变成"设置好跑起来"的系统（虾每天生成草稿，人每天确认发布）
