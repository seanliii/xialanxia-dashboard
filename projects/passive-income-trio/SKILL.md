# 被动收入三件套 — 复刻方法

## 这是什么
三条被动收入路径的计算器和规划工具：数字产品 + Newsletter + SaaS。输入你的参数（定价、受众规模、转化率），自动算出每条路径的预期月收入。

## 适合谁
- 想开始做被动收入但不知道选哪条路的人
- 正在纠结「做产品还是做内容还是做SaaS」的开发者
- 想量化评估不同被动收入模式可行性的人

## 复刻步骤

### Step 1：创建仓库
新建 GitHub 仓库 passive-income-trio，开启 Pages

### Step 2：三条路径设计

**路径1 - 数字产品**
- 输入：产品定价、预估月销量、平台抽成
- 输出：月净收入、年收入、breakeven 时间

**路径2 - Newsletter**
- 输入：订阅人数、打开率、广告CPM、付费转化率
- 输出：广告收入 + 付费收入 = 总月收入

**路径3 - Micro SaaS**
- 输入：月费定价、用户增长率、流失率、初始用户数
- 输出：12个月收入预测曲线

### Step 3：关键 Prompt

```
做一个被动收入三件套计算器页面：
- Tab切换三条路径：数字产品 / Newsletter / Micro SaaS
- 每条路径有独立输入表单和结果展示
- 数字产品：定价x销量-抽成=净收入
- Newsletter：订阅数x打开率xCPM + 付费人数x价格
- SaaS：MRR增长模型（含流失率），12个月预测
- 底部汇总：三条路径预估总月收入
- 深色主题，数据可视化
```

### Step 4：部署
推送代码，GitHub Pages 部署

## 踩坑记录
- SaaS 流失率对长期收入影响巨大，必须作为输入
- Newsletter 广告收入要区分 CPM 和 CPC 模式
- 数字产品的「月销量」初期很难预估，给个参考范围
