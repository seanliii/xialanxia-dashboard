# 内容创作者自动化 — 复刻方法

## 这是什么
一个人管多个自媒体平台的自动化助手。输入一个主题，自动按平台特性生成适配内容（公众号/小红书/Twitter/LinkedIn），批量生成省80%时间。

## 适合谁
- 一个人运营多个平台但精力不够的内容创作者
- 想做矩阵号但写不过来的自媒体人
- 需要跨平台分发内容的品牌营销

## 复刻步骤

### Step 1：创建仓库
新建 GitHub 仓库 content-creator-autopilot，开启 Pages

### Step 2：核心功能
- 主题输入：一个核心主题/观点
- 平台选择：勾选目标平台（公众号/小红书/Twitter/LinkedIn等）
- 一键生成：按每个平台的调性、字数、格式要求自动适配
- 预览对比：并排展示各平台生成结果
- 一键复制：每个平台内容可单独复制

### Step 3：关键 Prompt

```
做一个内容创作者自动化工具页面：
- 输入区：主题/核心观点（文本框）、选择目标平台（多选）
- 平台规则预设：
  公众号：2000字深度文、段落清晰、金句多
  小红书：300字+emoji密集+标签、种草风
  Twitter/X：280字以内、犀利观点、话题标签
  LinkedIn：专业调性、数据支撑、行业洞察
- 输出区：分栏展示各平台内容，支持复制
- 暗色科技感UI
```

### Step 4：部署
推送 main 分支，GitHub Pages 部署

## 踩坑记录
- 每个平台调性差异很大，不能简单缩写扩写
- 小红书需要 emoji 和口语化，公众号需要严谨深度
- Twitter 需要 hook 开头，前几个字决定生死
