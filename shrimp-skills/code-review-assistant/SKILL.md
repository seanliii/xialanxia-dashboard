# Code Review辅助虾

自动完成PR初次审核，人工只关注业务逻辑。

## 快速启用
配置references/code-standards.md（团队代码规范），虾自动检查每个PR并生成分级报告。

## 自动检查项
- 红色（必须修改）：SQL注入/安全漏洞/边界条件缺失
- 黄色（建议修改）：命名不规范/注释缺失/重复代码
- 绿色：符合规范

## 分工原则
- 虾负责：技术规范（命名/安全/注释）
- 人负责：业务逻辑（这里应该是>=而不是>）

## 踩坑避免
1. 误报太多：规范里写清楚「允许的缩写清单」
2. 虾越界判断业务逻辑：提示词明确「不要判断业务逻辑正确性」

## 验证成功标准
1个月后：因Code Review遗漏导致的线上bug数量=0。
