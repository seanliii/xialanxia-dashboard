# 三环分析 — 老张（SaaS 独立开发者）

## 模型说明

三环分析用于找到「能力 × 人群 × 渠道」的交集区域，交集越紧密，利基越可行。

---

## 环一：能力（What I Can Build）

| 维度 | 具体内容 |
|------|----------|
| 技术栈 | React + Python + PostgreSQL 全栈；可独立交付 Web SaaS |
| 产品能力 | 有完整 SaaS 上线经验（listing 优化工具已月入 $500） |
| 内容能力 | 英文技术博客 12 篇、SEO 基础、能写 landing page copy |
| 发行能力 | Product Hunt launch 两次（最高 800+ upvote）；熟悉 indie hacker 发行套路 |
| 边界 | 不能做需要多人协作的大型系统；不做中文市场 |

**能力关键词：** 全栈 SaaS / 数据抓取与处理 / SEO / 英文内容 / Product Hunt 发行

---

## 环二：人群（Who I Understand）

| 维度 | 具体内容 |
|------|----------|
| 主力人群 | 跨境电商卖家（曾为其开发 ERP 插件，深度理解工作流） |
| 次要人群 | 独立开发者 / Indie Hackers（自己就是其中一员） |
| 潜在人群 | 小型 DTC 品牌运营者（与跨境卖家高度重叠） |
| 人群痛点密度 | 跨境卖家日常大量重复操作（listing 翻译、竞品监控、review 管理、广告优化） |
| 关系深度 | 5 个付费客户 + ERP 插件时期积累的用户反馈 |

**人群关键词：** 跨境电商卖家 / 独立开发者 / 英文市场 B2B SaaS 用户

---

## 环三：渠道（Where I Can Reach Them）

| 渠道 | 覆盖力 | 转化潜力 | 说明 |
|------|--------|----------|------|
| Product Hunt | ★★★★★ | 高 | 两次 launch 经验，800+ upvote 证明能出圈 |
| 个人博客 | ★★★☆☆ | 中 | 月 5000 UV，SEO 长尾可持续获客 |
| Twitter (X) | ★★★☆☆ | 中 | 3000 英文粉，indie hacker 圈子活跃 |
| Discord 社群 | ★★★★☆ | 中高 | 3 个开发者群共 1500 人，可做早期验证 |
| 开源项目 README | ★★☆☆☆ | 低 | 800 star，可做导流入口 |

**渠道关键词：** Product Hunt / SEO 博客 / Twitter/X / Discord 社群
