# 验证实验计划 — ListingBridge

> 创始人：老张 | 日期：2026-05-18

---

## 实验目标

在 6 周内、0 开发成本的前提下，验证"多平台跨境卖家是否愿意为 AI Listing 本地化工具付费"。

---

## 实验设计

### 阶段一：需求验证（Week 1-2）

| 行动 | 预期结果 | 成功指标 |
|------|---------|---------|
| 发布 Landing Page（标题+价值点+waitlist表单） | 收集潜在用户 | 100+ 注册 |
| Product Hunt "Coming Soon" | 早期曝光+信号验证 | 50+ follows |
| Reddit r/FulfillmentByAmazon + r/ecommerce 发帖 | 需求讨论 + 获客 | 帖子 20+ upvotes |
| Facebook 跨境卖家群 3 个（英文）发调研问卷 | 痛点确认 | 30+ 回复 |
| Twitter 发布 building in public 系列 | 品牌曝光 | 5+ DM 咨询 |

### 阶段二：交付验证（Week 3-4）

| 行动 | 预期结果 | 成功指标 |
|------|---------|---------|
| 联系 waitlist Top 10，提供半价试用（$25/月） | 付费转化 | 5+ 付费 |
| 手动 + 脚本半自动交付（每单2小时内） | 交付质量 | 用户满意度 8+/10 |
| 每个用户做15分钟 exit interview | 产品反馈 | 明确的改进方向 |
| 记录每单交付耗时 | 规模化可行性 | < 30分钟/单（脚本优化后） |

### 阶段三：决策（Week 5-6）

| 情况 | 决策 |
|------|------|
| 5+ 付费 + NPS ≥ 8 + 续费意愿强 | ✅ 开始写 SaaS（6周 MVP） |
| 3-4 付费 + 反馈积极但有明确需求偏移 | ⚠️ 调整定位后再验证一轮 |
| < 3 付费 或 NPS < 6 | ❌ Pivot：考虑缩小到单一平台，或换方向 |

---

## 资源投入

| 资源 | 数量 | 说明 |
|------|------|------|
| 时间 | 20小时/周 × 6周 | 全职独立开发者，时间充裕 |
| 资金 | < $100 | Landing page 工具 + 域名 |
| 工具 | Carrd + Mailchimp + GPT API + Python | 已有或免费 |
| 渠道 | PH + Reddit + Twitter + FB Groups | 已有账号和信任度 |

---

## 风险及缓解

| 风险 | 概率 | 缓解措施 |
|------|:----:|---------|
| Landing page 没人来 | 中 | 多渠道推广 + 个人品牌背书 |
| 注册多但不付费 | 中 | 半价降低决策门槛 + 退款保证 |
| 手动交付质量不够好 | 低 | 老张全栈能力 + GPT质量高 |
| 竞品突然出现 | 低 | 6周验证窗口足够快 |
| 用户要的不是翻译而是其他功能 | 中 | interview 快速捕捉真实需求 |
