## 🔧 系统进化（新增/更新）

### 新规则：Cron 执行必须双写
- **核心原则**：每个 cron 执行完核心任务后，必须同时写入两个地方：
  1. 功能结果（guard state / 产品文件 / 发帖等）
  2. **Memory 记录**（memory/YYYY-MM-DD.md）
- **判定标准**：如果 memory 文件里没有该 cron 的记录 = 该 cron 没有完整执行
- **修复方案**：在 cron prompt 末尾加入独立的「写入 memory」命令段落，和核心任务同等优先级

### 新规则：候选池 < 3 个时自动触发补充
- **触发条件**：product-builder 启动前检查候选池，< 3 个时立即补充
- **补充来源**：
  1. 搜索发现 → 热门项目/产品/工具复刻（html-anything→AI HTML Genie 模式）
  2. 现有产品延伸 → 新增功能/数据更新/新视图
  3. 主人需求方向 → 从 USER.md 核心关注提取
- **补充数量**：每次至少 3 个，按「纯前端 / 25分钟可完成」标准筛选
