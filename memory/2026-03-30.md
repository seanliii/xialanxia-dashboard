## 15:11 — Dashboard 彻底修复

### 根本原因
- 当前 `dashboard.html` 的 `</div>` 比 `<div>` **多 183 个**（历史 inject_block bug 长期累积）
- 多余闭合标签提前关闭了 `.shell` 容器，导致所有 `.page` div 跑到 `.shell` 外面
- CSS 选择器 `#tab-xxx:checked ~ .shell #page-xxx` 因此永远失效
- 结果：每个 tab 点击都显示相同内容（实为泄漏到 body 顶层的默认页）

### 修复方案
1. 从 git 历史找到最近 div 完全平衡的干净版本（commit `903f6f3`，100.2KB）
2. 用 depth-counting 提取当前文件中的今日数据块（today-stocks/today-intel/today-evolution/today-memory）
3. 将这些块注入到干净基础版本
4. 验证结果：div balance=0，全部 11 个 page 存在，12 个 radio input 正常
5. 推送 → https://seanliii.github.io/xialanxia-dashboard/

### 教训
- `inject_block` 的 `.*?` 非贪婪 regex bug 不只影响 `today-stocks`，还污染了多个块
- 今后需在每次 inject 后立即验证 div balance，不能等累积到上百个才发现
- 有 git 历史在手，损坏的 HTML 总有办法还原
