## 🔧 系统进化（今日新增）

### guard 写入失效：深度原因 + 修复方向
- **深度原因**：LLM session 是无状态的，prompt 末尾的「最后一步」依赖 LLM 自愿执行
- **已知规律**：脚本型任务（project-verify / memory-compress）= 100% 写入；LLM型任务 = 不稳定
- **修复方向**：
  1. Prompt强化版：在 cron 定义里用更强的强制语气（"任务未完成直到运行done"）
  2. 包裹层：给 LLM cron 加一个外层守护脚本，session 结束后自动检查并写 done
  3. 降级：接受 LLM cron guard 不可靠，在巡检里加「软故障」提醒而不是依赖它

**今日选择方案**：方案3（接受现实，加告警）+ 强化 Prompt（方案1）同步执行
