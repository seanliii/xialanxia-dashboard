# Behavioral Profile

## Security and Installation Practices

- User strongly prioritizes a security-first and cautious approach to AI agent installation, execution, and script management.
- User strictly rejects running any unverified installation scripts such as 'curl | bash' without prior thorough auditing and verification.
- User manages API credentials carefully to prevent security risks and operational failures.
- User favors using fully functional APIs that minimize sharing sensitive credentials and avoid complex additional authentication.
- User clarifies that external access to the project-openclaw dashboard on port 9876 depends on environment port mappings.

## AI Agent Execution Preferences

- User prefers running only a single AI agent per physical machine to minimize security risks, resource conflicts, concurrency issues, and to promote resource efficiency.
- User avoids concurrent multi-agent execution on a single machine, favoring minimal concurrency.
- User prioritizes depth of architectural integration and evolutionary AI system design over the number of agents running concurrently.

## Architectural and Research Interests

- User is deeply interested in evolutionary AI agent architecture.
- User actively integrates open-source architectural designs, notably from project-hermes-agent, into their own project, project-openclaw.
- User emphasizes deep architectural understanding and integration rather than concurrent multi-agent execution.
- User favors fully functional APIs without complex additional authentication for external integrations.

## Knowledge and Memory Management

- User employs methodical and consistent automated tools for effective knowledge management.
