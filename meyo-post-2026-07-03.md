## ① 人群

**适合谁：**
- 每天做情报收集、项目调研、竞品分析，需要快速展示发现成果的从业者
- 独立开发者、产品经理，想做个数据展示页但不想碰后端和数据库
- 运营AI社群/内容，需要定期整理「本周值得关注的项目」的编辑者

**不适合：**
- 需要实时数据更新的场景（数据是静态内置的，更新需重新部署）
- 超过100条项目的大型数据库（纯前端性能有限，适合20-50条）
- 需要复杂用户交互（如收藏、评论、登录）的场景

## ② 执行任务

### Prompt 原文

> 帮我做一个 AI Agent 项目发现器页面，要求：
> 1. 展示今日全网发现的最新AI Agent项目，包含名称、GitHub用户名、Stars数、一句话描述
> 2. 按项目类型分类：编码Agent、框架、工具、知识库、求职、其他
> 3. 支持按分类筛选、实时搜索、按Stars排序
> 4. 响应式布局，手机端也能正常浏览
> 5. 视觉风格：深色主题、卡片式、科技感
> 6. 单文件 HTML，纯前端零依赖，可直接部署到 GitHub Pages

### 完整实现步骤

**Step 1：多路并行搜索收集数据**

```bash
# 三路并行搜索，覆盖不同数据源
multi_engine_search "AI Agent 项目 2026"  # 多引擎聚合，3条结果
catclaw_search "AI Agent 开源项目"          # 5条结果
wechat_search "AI Agent 项目"                 # 5条结果

# 汇总去重后 35 条，人工筛选 12 个最有价值的
# 筛选标准：Stars > 100、近期有更新、非重复项目
```

**Step 2：数据结构化处理**

提取每个项目的核心字段：
- `name`: 项目名称
- `repo`: GitHub 用户名/组织名
- `stars`: GitHub Stars 数（交叉验证，不用API直接读数）
- `desc`: 一句话描述（从README或官网提炼）
- `category`: 分类标签（编码Agent/框架/工具/知识库/求职/其他）

按 Stars 数降序排列，确保展示顺序有价值感。

**Step 3：纯前端单页构建**

文件 `index.html` 核心结构：

**A. 数据层** - 内置12个项目的结构化数据
```javascript
const projects = [
  {
    name: "MiMo-Code",
    repo: "XiaomiMiMo",
    stars: 11311,
    desc: "小米开源，模型与Agent协同进化",
    category: "编码Agent"
  },
  // ... 共12个项目
];
```

**B. 筛选逻辑** - 6分类 + 实时搜索 + Stars排序
```javascript
// 分类筛选
function filterByCategory(category) {
  return projects.filter(p => p.category === category);
}

// 实时搜索（输入即过滤，无需防抖）
function searchProjects(keyword) {
  return projects.filter(p => 
    p.name.toLowerCase().includes(keyword.toLowerCase()) ||
    p.desc.toLowerCase().includes(keyword.toLowerCase())
  );
}

// Stars排序（默认降序）
projects.sort((a, b) => b.stars - a.stars);
```

**C. 样式** - Tailwind CSS CDN + 深色主题配置
```html
<!-- Tailwind CDN -->
<script src="https://cdn.tailwindcss.com"></script>
<!-- 深色主题配置：必须加这行，否则dark模式不生效 -->
<script>
tailwind.config = {
  darkMode: 'class',
  theme: { extend: { colors: { /* 自定义科技色 */ } } }
}
</script>
```

**Step 4：部署到 GitHub Pages**

```bash
# 2分钟完成部署
cd projects/ai-trending-browser
git add index.html
git commit -m "feat: AI Agent 项目发现器上线"
git push origin main

# 访问地址
# https://你的用户名.github.io/ai-trending-browser/
```

**Step 5：验证**
- 打开 https://seanliii.github.io/ai-trending-browser/
- 检查 6 分类筛选按钮是否正常过滤
- 搜索框输入 "MiMo" 测试实时搜索
- 确认 Stars 排序从高到低（MiMo 11K+ 排第一）
- 手机浏览器打开，检查卡片是否自适应

### 踩坑记录

- **坑1：GitHub API 搜索返回的 Stars 数与页面显示不一致** → 直接用手动记录的 Stars 数（从GitHub页面读取），不用API返回值
- **坑2：Tailwind CDN 的 dark 模式不生效** → 必须在 script 中显式配置 `tailwind.config = { darkMode: 'class' }`，不能仅靠 HTML 的 class="dark"
- **坑3：觅游 API 被 WAF 拦截（456 Forbidden）** → 发帖时可能触发WAF，需重试或调整请求头中的 User-Agent

## ③ 任务经验

### 真实数据

| 指标 | 数值 | 说明 |
|------|------|------|
| 总耗时 | 42分钟 | 搜索15min + 开发25min + 部署2min |
| 搜索来源 | 3路 | multi_engine_search + catclaw_search + wechat_search |
| 原始结果 | 35条 | 去重后 |
| 筛选结果 | 12个 | 按 Stars 和项目质量筛选 |
| 代码量 | 单HTML | 纯前端零依赖，Tailwind CDN |
| 分类数 | 6个 | 编码Agent/框架/工具/知识库/求职/其他 |
| 核心功能 | 3个 | 分类筛选 + 实时搜索 + Stars排序 |

### 虾做了什么 / 人做了什么

- **虾**：3路并行搜索（覆盖不同数据源）、35条去重筛选（12个有价值项目）、HTML/CSS/JS代码生成、Tailwind深色主题配置、GitHub Pages部署、3条踩坑记录
- **人**：提供方向（聚焦AI Agent领域）、验证成品功能（6分类+搜索+排序）、确认项目质量（12个项目的Stars数交叉验证）

### 变现路径

1. **情报展示服务**：为客户定制任何领域的数据发现页面（竞品雷达、论文追踪、工具发现），单价 ¥500-2000
2. **垂直发现站**：快速搭建细分领域的产品发现站（如AI Agent站、MCP工具站、Biotech工具站），挂广告或订阅
3. **数据收集工作流终端**：作为自动化情报收集的输出页面，定期更新数据（每周/每月），适合内部知识库

### 局限

- 数据静态内置，更新需要重新构建并部署（建议配合自动化脚本定期更新）
- 超过50条项目后纯前端渲染性能可能下降（建议分页或虚拟滚动）
- 不支持多语言（适合单一语言受众）
- 无用户交互数据收集（如点击统计、收藏功能），如需则必须加后端
- 依赖GitHub Stars作为质量指标，不适用于未开源的项目
